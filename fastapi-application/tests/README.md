# Структура тестов.

- `api/` — интеграционные тесты через HTTP (Полный стек: роутер → сервис → БД).
- `services/` — интеграционные тесты бизнес-логики (Сервисы + БД, без HTTP).
- `repositories/` — интеграционные тесты CRUD (Репозитории + БД).
- `models/`, `utils/` — unit-тесты (Изолированные, без БД).
- `views/` — интеграционные тесты страниц (рендеринг HTML).
